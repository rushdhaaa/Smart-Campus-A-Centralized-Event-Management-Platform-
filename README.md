## вқ— Problem Statement

Campus event management in most colleges is chaotic and fragmented. Events are announced on notice boards, registrations happen via WhatsApp groups, and there is **zero centralized tracking** of attendance or feedback.

| Pain Point | Problem |
|---|---|
| рҹ“ӯ **No Central Platform** | Events are scattered across emails, posters, and social media. Students miss events they would have attended. |
| рҹ“қ **Manual Registrations** | Paper-based or spreadsheet-driven registration has no capacity enforcement, duplicate detection, or real-time availability. |
| рҹ“Ҡ **Zero Admin Visibility** | Admins cannot filter events by department, date, or type. No dashboards showing registration counts or statistics. |
| рҹ”Ү **No Feedback Loop** | After events, there is no structured way to collect student ratings, making it impossible to improve future events. |
| рҹӨ– **No Personalization** | Students must manually browse all events with no intelligent filtering based on their interests. |

---

## вң… Our Solution

**Smart Campus Event Management System** is a full-stack web application built with Spring Boot 3 that provides a unified platform for the entire campus event lifecycle вҖ” from discovery to feedback.

```
Student Journey:   Landing вҶ’ Login вҶ’ Browse Events вҶ’ AI Recommendations вҶ’ Register вҶ’ View My Events вҶ’ Submit Feedback
Admin Journey:     Landing вҶ’ Login вҶ’ Admin Panel вҶ’ Add/Edit/Delete Events вҶ’ View Stats & Filters
```

| Solution | Description |
|---|---|
| рҹ–ҘпёҸ **Unified Event Hub** | Single dashboard with all upcoming events вҖ” type badges, capacity bars, venue, and real-time seat availability |
| рҹ”’ **Smart Registration Engine** | Server-side rules prevent duplicates, enforce capacity limits, and reject registrations for invalid event states |
| вҡҷпёҸ **Admin Control Panel** | Secured with HTTP Basic Auth вҖ” full CRUD, multi-field filtering, date range search, and live statistics |
| вӯҗ **Structured Feedback** | 5-star ratings + written comments per event; admins can retrieve average ratings via REST API |
| рҹӨ– **AI Recommendations** | Google Gemini AI scores every event against student interests and returns a ranked list with match reasons |

## рҹӨ– AI Recommendation System

### How It Works

The recommendation engine uses **Google Gemini 2.5 Flash** to intelligently score every event in the database against a student's free-text interest string.

```
Student types interest  вҶ’  RecommendationController
        в”Ӯ
        в–ј
RecommendationService.getRecommendedEvents(interest)
        в”Ӯ
        в”ңв”Җв”Җ Fetch all events from EventRepository
        в”Ӯ
        в”ңв”Җв”Җ Build a single prompt with all event titles, types & descriptions
        в”Ӯ
        в–ј
Google Gemini API (v1beta В· gemini-2.5-flash)
  POST https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent
        в”Ӯ
        в–ј
  Returns JSON array:  [{ id, score, reason }, ...]
        в”Ӯ
        в”ңв”Җв”Җ Parse scores вҶ’ map to Event entities
        в”ңв”Җв”Җ Assign MatchLevel: HIGH / MEDIUM / LOW
        в”ңв”Җв”Җ Sort by score DESC
        в””в”Җв”Җ Return ranked RecommendationResult list
                в”Ӯ
                в–ј
        RecommendationController
        вҶ’ JSON response to dashboard.html
        вҶ’ Rendered as neon recommendation cards
```

### Single API Call Design

All events are sent in **one prompt** to Gemini, making it cost-efficient regardless of how many events exist. Gemini scores all events in one shot and returns a ranked JSON array.

### Configuration

Set the following in `application.properties`:

```properties
# в”Җв”Җ Google Gemini Configuration в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
# Get your free key from: https://aistudio.google.com/app/apikey
gemini.api.key=YOUR_API_KEY_HERE
gemini.api.model=gemini-2.5-flash
```

> вҡ пёҸ **Important:** Use model `gemini-2.5-flash` with API version `v1beta`. Other combinations (`gemini-2.0-flash` on `v1`, `gemini-1.5-flash-8b` on `v1`) result in 404 errors. The free tier quota for `gemini-2.5-flash` on `v1beta` is generous вҖ” up to 1 million tokens/day.

### Match Level Thresholds

| Score Range | Match Level | Meaning |
|---|---|---|
| 65 вҖ“ 100 | рҹҹў HIGH | Strongly aligned with student interests |
| 35 вҖ“ 64 | рҹҹЎ MEDIUM | Partially relevant |
| 0 вҖ“ 34 | рҹ”ҙ LOW | Little to no relevance |

Events with a score of 0 are excluded from results entirely.

### Fallback Behavior

If the Gemini API call fails for any reason (network error, quota exceeded, invalid key), the service returns an empty list gracefully and the UI shows "Recommendations not available. Please try again later." вҖ” the rest of the application continues working normally.

## вҡ пёҸ Important Notes

### рҹ”§ Why Lombok Was Removed

The original version of this project used Lombok annotations:
```java
// ORIGINAL вҖ” breaks without Annotation Processing enabled
@Data
@Slf4j
@Builder
@RequiredArgsConstructor
public class Event { ... }
```

Without enabling **Annotation Processing** in IntelliJ IDEA, Lombok silently generates nothing. This caused **49 compilation errors** like:
```
cannot find symbol: method getTitle()
cannot find symbol: variable log
cannot find symbol: method builder()
variable eventService not initialized in the default constructor
```

**This fixed version removes Lombok entirely** and uses explicit Java:

```java
// FIXED вҖ” works everywhere, zero tool configuration needed
public class Event {
    private String title;

    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }
}

// Explicit logger
private static final Logger log = Logger.getLogger(EventService.class.getName());

// Explicit constructor injection
public EventService(EventRepository eventRepository) {
    this.eventRepository = eventRepository;
}
```

- **Free Quota:** ~1 million tokens/day, 15 requests/minute вҖ” sufficient for normal use
- **Key Security:** Never commit your API key to Git. Add `application.properties` to `.gitignore` or use environment variables in production
- **Rate Limits:** If you hit 429 errors, wait 60 seconds and try again, or generate a new key from AI Studio

---

### рҹ’ҫ H2 is In-Memory вҖ” Data Resets on Restart

H2 stores data **only while the application is running**. Everything is lost when the server stops.

- вң… The 8 sample events are **re-inserted from `data.sql` automatically** on every startup
- вқҢ Student registrations and feedback are lost on restart

**To use persistent storage**, switch to MySQL in `application.properties`:
```properties
# Replace H2 config with:
spring.datasource.url=jdbc:mysql://localhost:3306/campusdb
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.username=root
spring.datasource.password=yourpassword
spring.jpa.hibernate.ddl-auto=update
```
And add MySQL dependency to `pom.xml`:
```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

## рҹҗӣ Common Issues & Fixes

### вқҢ Port 8080 Already in Use
```
Error: Web server failed to start. Port 8080 was already in use.
```
**Fix:** Change the port in `src/main/resources/application.properties`:
```properties
server.port=8090
```
Then access the app at `http://localhost:8090`.

---

### вқҢ `mvn` is not recognized
```
'mvn' is not recognized as an internal or external command
```
**Fix:** Maven is not added to PATH. Either:
- Add Maven's `bin` folder to your system PATH, or
- Use the full path: `C:\apache-maven-3.x.x\bin\mvn spring-boot:run`

---

### вқҢ `java -version` shows version below 17
```
java version "11.0.x"
```
**Fix:** Install Java 17+ from [adoptium.net](https://adoptium.net) and set `JAVA_HOME` to the JDK 17 installation directory.

---

### вқҢ H2 Console shows "jdbc:h2:~/test" (wrong URL)
**Fix:** Change the JDBC URL in the H2 console login page to:
```
jdbc:h2:mem:campusdb
```

---

### вқҢ Admin panel says "Unauthorized" or loops on Basic Auth
**Fix:** Clear your browser's cached credentials for `localhost:8080`, then re-enter:
- Username: `admin`
- Password: `admin123`

---

### вқҢ Sample data not loading (events table empty)
**Fix:** Check `application.properties` has:
```properties
spring.sql.init.mode=always
spring.jpa.hibernate.ddl-auto=create-drop
```
Both must be set correctly for `data.sql` to run on startup.

---

### вқҢ AI Recommendations show "Recommendations not available"
```
Gemini call failed вҖ” returning empty recommendations.
```
Check the Spring Boot console for the exact reason. Common causes and fixes:

| Error | Fix |
|---|---|
| `404 NOT_FOUND` вҖ” wrong model name | Use `gemini.api.model=gemini-2.5-flash` in `application.properties` |
| `429 RESOURCE_EXHAUSTED` вҖ” quota exceeded | Generate a new API key at [aistudio.google.com/app/apikey](https://aistudio.google.com/app/apikey) |
| `400 BAD_REQUEST` вҖ” malformed request | Ensure `v1beta` is in the URL (not `v1`) |
| Empty `gemini.api.key` | Paste your actual key in `application.properties` |
| JSON parse error / truncated response | Already fixed вҖ” `maxOutputTokens` is set to `8192` |
